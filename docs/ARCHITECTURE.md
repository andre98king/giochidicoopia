## 1. Vision & Scope

**CoopHubs** (`coophubs.net`) is a curated static catalog of PC co-op games. It is designed as a "zero-backend" SEO engine that serves both human visitors (via a SPA-like client experience) and search crawlers (via pre-rendered static pages with Schema.org structured data).

**Core value proposition**: the place gamers go when they want to say "let's play together tonight" — a curated, quality-certified PC co-op catalog.

**Current scale**:
- **446 games** in the catalog (406 static IT pages + 406 static EN pages)
- **8 IT hub pages** + **8 EN hub pages** for strategic SEO keywords
- **~900 KB** data bundle transferred on first load
- **~5-30 min** CI workflow duration
- **43 Python scripts** in the pipeline (2 removed in 2026-04-30 cleanup)
- **43 Python scripts** in the pipeline (2 removed in 2026-04-30 cleanup)

## 4. Frontend Architecture

### 4.1 Dual-Mode Rendering

The frontend has two complementary modes:

1. **SPA Runtime** (`index.html` + `app.js`): loads game data dynamically and renders cards with filters, search, and modals. This is the human-facing experience.
2. **Static SEO Pages** (`games/<id>.html`): pre-rendered HTML with meta tags, JSON-LD, hreflang, and canonical links. These exist solely for search crawlers and direct-link visitors.

### 4.2 Data Loading Strategy (app.js)

`app.js` attempts to load catalog data in this priority order:

1. **`fetch(data/catalog.public.v1.json)`** — preferred source (~400 KB JSON, only frontend fields)
2. **Legacy fallback: `loadLegacyCatalogScript()`**
   - Injects `assets/games.js` (loader)
   - Loader injects `assets/bundles/games-data.js` (data)
   - Uses `window.games` global array
3. If all fail, renders empty grid with console warning

```javascript
// From app.js — loadCatalogData()
const response = await fetch(`data/catalog.public.v1.json?v=${cacheBucket}`);
const payload = await response.json();
catalogGames = payload.games;
```

### 4.3 Bundle Split: games.js vs games-data.js

| File | Size | Role |
|------|------|------|
| `assets/games.js` | 2.4 KB | Async loader. Defines `loadGamesData()`, `getGames()`, `featuredIndieId` |
| `assets/bundles/games-data.js` | ~900 KB | `const games = [ { id:..., title:... }, ... ]` (446 entries) |

**Why the split?** The original monolithic `games.js` was ~1.1 MB. The split allows the tiny loader to be inlined/cacheable while the data bundle can be lazy-loaded or cached separately.

**⚠️ Risk**: Several scripts still reference the monolithic path (`assets/games.js`) for regex-based edits. See Section 9 (Risks).

### 4.4 Key Frontend Features

| Feature | Implementation |
|---------|---------------|
| **Filters** | AND-logic on `categories`, `coopMode`, `maxPlayers`, `crossplay`, `releaseYear` |
| **Search** | Text match on `title`, `description`, `description_en`, `categories` |
| **Sorting** | `default`, `rating`, `trending`, `newest`, `az` |
| **Modal** | `openModal(id)` — dynamic HTML injection with focus trap |
| **Random Wheel** | Canvas-based spinner, samples from currently filtered games |
| **Played/Notes** | `localStorage` keys: `coophubs_played`, `coophubs_notes` |
| **Free Games** | Async load of `assets/free_games.js` with 4s timeout |
| **i18n** | `t(key)` function, `setLang('it'|'en')`, `data-i18n` attributes |

### 4.5 Internationalization

- Default language: **IT**
- Toggle button switches `currentLang` and re-renders UI
- All user-facing strings live in `assets/i18n.js`
- Static pages have separate IT/EN versions generated by `build_static_pages.py`

## 5. Data Layer

### 5.1 Game Object Schema

Each game in the catalog is a JavaScript object with these fields:

```javascript
{
  id: 1,                    // Integer, unique, sequential (not reused)
  igdbId: 12345,            // IGDB game ID (0 if unknown)
  title: "Game Title",
  categories: ["action", "rpg"],    // Site taxonomy (max 3)
  genres: ["action", "rpg"],        // Raw genres from sources
  coopMode: ["online"],             // Canonical: "online" | "local" | "sofa"
  maxPlayers: 4,
  crossplay: false,
  players: "1-4",           // Human-readable label
  releaseYear: 2025,
  image: "https://cdn.cloudflare.steamstatic.com/...",
  description: "Italian description...",
  description_en: "English description...",
  personalNote: "",         // Editorial note (not displayed in public)
  played: false,            // Editorial tracking
  steamUrl: "https://store.steampowered.com/app/12345/...",
  gogUrl: "",
  epicUrl: "",
  itchUrl: "",
  ccu: 15000,               // Current concurrent players on Steam
  trending: true,           // true if ccu >= MIN_CCU_TRENDING (10,000)
  rating: 87,               // Steam review score 0-100
  igUrl: "...", igDiscount: 15,
  gbUrl: "...", gbDiscount: 10,
  gsUrl: "...", gsDiscount: 0,
  kgUrl: "...", kgDiscount: 0,
  k4gUrl: "...", k4gDiscount: 0,
  gmvUrl: "...", gmvDiscount: 0,
  gmgUrl: "...", gmgDiscount: 0,
  coopScore: 2,             // 1=marginal, 2=solid, 3=co-op core, null=unscored
  mini_review_it: "",
  mini_review_en: "",
}
```

### 5.2 Catalog Files

| File | Size | Contents | Written By | Read By |
|------|------|----------|-----------|---------|
| `assets/bundles/games-data.js` | ~900 KB | JS array of 446 games (all fields) | `write_legacy_games_js()` | `catalog_data.load_games()`, browser |
| `assets/games.js` | 2.4 KB | Loader + `featuredIndieId` | `write_legacy_games_js()` | Browser (injected script) |
| `data/catalog.games.v1.json` | ~1.6 MB | Full catalog with **derived fields** | `write_catalog_artifact()` | Pipeline scripts |
| `data/catalog.public.v1.json` | ~900 KB | Public subset (no private/editorial fields) | `write_public_catalog_export()` | `app.js` (preferred), humans |

### 5.3 Derived Fields (Normalize)

`catalog_data.normalize_game()` adds these fields on every load:

- `slug` — URL-safe slug from title
- `isFeaturedIndie` — boolean, matches `featuredIndieId`
- `descriptions` — `{it, en}` object (redundant with flat fields)
- `taxonomy` — `{categories, genres}` object
- `capabilities` — `{onlineCoop, localCoop, splitScreen, sharedScreen, crossplay, playersLabel, maxPlayers}`
- `storefronts` — array of `{store, url, externalId}`
- `externalIds` — `{steamAppId, epicSlug, itchSlug}`
- `signals` — `{ccu, trending, rating, igdb, gog, rawg}`
- `editorial` — `{played, personalNote}`
- `sourceMeta` — `{ingestSource, schemaVersion}`

**⚠️ These derived fields bloat `catalog.games.v1.json` and `games-data.js`.** The JSON artifact is 1.6 MB despite the raw data being ~900 KB. Reducing the serialized field set in `write_legacy_games_js()` would cut the bundle size by ~40%, but requires coordination with the CI pipeline (see Risks).

## 7. CI/CD Workflows

### 7.1 `.github/workflows/update.yml` — Daily 06:00 UTC

The main pipeline. Runs every day at 06:00 UTC. Has **7 jobs**:

```
Job 1: auto-update-db
  └── auto_update.py
        ├── Load existing catalog (games-data.js)
        ├── Update: CCU, rating, trending, releaseYear
        ├── Discovery: Steam new releases, IGDB, GOG, RAWG, itch.io
        ├── Quality gate (quality_gate.py)
        ├── Enrich (SteamSpy, descriptions, categories)
        └── Write: games.js + games-data.js + catalog.*.json
        └── Upload artifact: db-output

Jobs 2-6 (parallel, continue-on-error):
  fetch-affiliate  → fetch_affiliate_prices.py  → artifact: affiliate-games
  fetch-gameseal   → fetch_gameseal_prices.py   → artifact: gameseal-games
  fetch-gamivo     → fetch_gamivo_prices.py     → artifact: gamivo-games
  fetch-k4g        → fetch_k4g_prices.py        → artifact: k4g-games
  fetch-gmg        → fetch_gmg_prices.py        → artifact: gmg-games
  Each: download db-output → update prices → upload artifact

Job 7: build-and-deploy
  ├── Download db-output
  ├── Download all 5 affiliate artifacts → data/artifacts/
  ├── merge_affiliate_artifacts.py
  ├── build_static_pages.py
  ├── build_hub_pages.py
  ├── validate_catalog.py
  └── git commit + push
```

**Artifacts**: `db-output`, `affiliate-games`, `gameseal-games`, `gamivo-games`, `k4g-games`, `gmg-games`.

### 7.3 `.github/workflows/catalog_audit.yml` — Weekly Sunday 03:00 UTC

- `catalog_audit.py --fast --apply` — audits trending games, auto-removes false co-op entries

## 9. SEO Architecture

### 9.1 Static Game Pages

For every game, two static pages are generated:
- `games/<id>.html` (Italian)
- `games/en/<id>.html` (English)

Each page contains:
- `<title>`: max 60 chars, format `"{title} ({coop_str} · {players}P) | GiochiDiCoop"`
- `<meta name="description">`: max 155 chars, first sentence of description + CTA
- `<link rel="canonical">` + `<link rel="alternate" hreflang="...">` (bidirectional IT↔EN)
- **Schema.org `VideoGame` JSON-LD** — generated by `seo_content_generator.py`
- **Thin content block**: deterministic 130-170 words, seeded by `hashlib.md5(game_id)`
- Preconnect + DNS-prefetch hints for Steam CDN
- Hero image with `fetchpriority="high"`

### 11.2 Frontend Optimizations

- **Font preloading**: Space Grotesk + JetBrains Mono via `<link rel="preload" as="font">` with `crossorigin`
- **Preconnect**: `shared.cloudflare.steamstatic.com` (Steam CDN)
- **DNS-prefetch**: `store.steampowered.com`
- **Hero image**: `fetchpriority="high"`, no `loading="lazy"`
- **Lazy loading**: All other images use `loading="lazy"` + `decoding="async"`
- **Cache busting**: `?v=20260327` on CSS/JS assets
- **GDPR**: No Google Fonts CDN, no analytics, no cookies

## 12. Security & Constraints

### 12.1 Architectural Constraints (Hard Rules)

1. **No backend server** — Zero Express, Flask, FastAPI, Node. Sito 100% statico.
2. **No npm/package.json** — No build step, no bundler, no dependency hell.
3. **No analytics** — GDPR choice. No Google Analytics, no tracking pixels.
4. **No cookies** — Only `localStorage` for played/notes state.
5. **Routing ID-based** — `games/{id}.html`. **Never** switch to slug-based routing.
6. **Idempotency** — Every write script checks `if content == existing: continue`.
7. **Safe templating** — Use `safe_template()` or `.replace()`, never f-strings in HTML blocks.

### 12.2 API Keys & Secrets

Stored in GitHub Secrets and passed as env vars in workflows:
- `STEAM_API_KEY`
- `IGDB_CLIENT_ID`, `IGDB_CLIENT_SECRET`
- `RAWG_API_KEY`
- `CJ_API_TOKEN`
- `ITCH_IO_KEY`

## 13. Risks & Anti-Patterns to Avoid

### 13.1 The Bundle Split Inconsistency

Several scripts were written when `games.js` was a monolithic file containing all game data. After the bundle split:

- **`remove_game.py`** — Uses regex to find/remove game blocks from `assets/games.js`. Since `games.js` is now only the loader (2.4 KB), this script **cannot find game data** and will fail or corrupt the file.
- **`catalog_ingest.py`** — Same issue: appends new games to `assets/games.js` via regex insertion. Will insert into the loader file, breaking it.
- **`merge_affiliate_artifacts.py`** — Points to `assets/bundles/games-data.js` (correct), but uses regex parsing that may be fragile.

**Mitigation**: `add_classics.py` and `apply_fixes.py` were updated to use `catalog_data.load_legacy_catalog_bundle()` and `write_legacy_games_js()`, so they work correctly.

### 13.2 Derived Field Bloat

`normalize_game()` adds 11 derived fields every time `load_games()` is called. These fields are then serialized back into `games-data.js` and `catalog.games.v1.json` by `write_legacy_games_js()`, even though they are never used by the frontend.

**Impact**: `catalog.games.v1.json` is 1.6 MB instead of ~900 KB. `games-data.js` is ~900 KB instead of ~500 KB.

**Fix (future work)**: Modify `write_legacy_games_js()` to serialize only the 30 fields actually used by `app.js`, or add a separate "frontend-only" serializer.

### 13.3 CI Auto-Commit Risk

The CI commits directly to `main` with force-push. A bug in `auto_update.py` or a fetcher script could corrupt `games-data.js` and auto-commit the corruption.

**Mitigation**: `validate_catalog.py` runs after every build and exits with code 1 on critical errors (duplicate IDs, missing pages, suspiciously small catalog). This blocks the commit.

### 13.4 Rate Limiting

The CI runs daily and queries multiple external APIs. Key limits:
- Steam API: ~60/min (DELAY = 1.5s)
- IGDB: ~120/min (0.5s delay)
- RAWG: generous but requires key
- CJ Affiliate: token-based, rate limits vary

Scripts use `catalog_config.DELAY` and `continue-on-error` for the affiliate jobs to avoid blocking the pipeline.

## 14. Common Tasks (How-To)

### Add a new game manually

```bash
# 1. Prepare the game entry as JSON in data/new_game.json
# 2. Run the deprecated but functional ingest
python3 scripts/catalog_ingest.py --input data/new_game.json --apply
# 3. Rebuild static pages
python3 scripts/build_static_pages.py
python3 scripts/build_hub_pages.py
# 4. Validate
python3 scripts/validate_catalog.py
```

### Update a game's data (bulk fix)

```bash
# 1. Edit data/pending_fixes.json
# 2. Dry-run
python3 scripts/apply_fixes.py
# 3. Apply
python3 scripts/apply_fixes.py --apply
# 4. Rebuild + validate
python3 scripts/build_static_pages.py
python3 scripts/validate_catalog.py
```

### Run the full CI pipeline locally

```bash
# 1. Set env vars (or have .env file)
export STEAM_API_KEY=xxx
export IGDB_CLIENT_ID=xxx
# ... etc

# 2. Run update
python3 scripts/auto_update.py

# 3. Run curation gate
python3 scripts/run_curation_gate.py

# 4. Fetch prices (optional, slow)
python3 scripts/fetch_affiliate_prices.py

# 5. Build pages
python3 scripts/build_static_pages.py
python3 scripts/build_hub_pages.py

# 6. Validate
python3 scripts/validate_catalog.py
```

### Test the frontend locally

Simply open `index.html` in a browser. No server required — it's all static. For CORS-free testing of `fetch(data/catalog.public.v1.json)`, serve via any static server:

```bash
python3 -m http.server 8000
# Then open http://localhost:8000
```
